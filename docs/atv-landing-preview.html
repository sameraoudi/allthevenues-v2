How this page works
• Route: /locations; the "Locations" nav link (removed earlier) comes back pointing here.
• One tile per emirate from the emirates table, rendered only when it has ≥1 published venue (empty emirates are hidden). Each tile links to /venues?emirate={slug}. A quick-jump chip row under the intro echoes the tiles.
• Cover = a provided city image at assets/img/locations/{emirate-slug}.webp (Samer supplies these). Fallback order: city image → a top venue photo from that emirate → seeded gradient. Placeholders shown here.
• Al Ain stays out of the emirate mosaic (it's a city within Abu Dhabi) — reserved for a future "Popular areas / Browse by area" section, not forced into the emirate layout.
• Count rule mirrors Event Types: a number when it's reasonably strong, else "Explore venues" (Ajman / Fujairah shown that way here).
• Mosaic hierarchy = Dubai hero, Abu Dhabi tall, the rest quads (by venue volume); hover reveals the descriptor on quads (desktop), headings-only on mobile — consistent with the Event Types page.
