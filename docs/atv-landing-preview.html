How this page works
• Route: /venues/{event}-in-{emirate}, e.g. /venues/wedding-in-dubai. The router checks real venue slugs first, then this -in- pattern; the event + emirate slugs are validated against the taxonomy (invalid → 404).
• Selective, not exhaustive: we do NOT auto-publish every event×city combo. A landing page renders only when the combo has ≥ a threshold of published venues; a below-threshold combo 301-redirects to the filtered venue search (/venues?event_type=…&emirate=…) instead of showing a thin page. Only qualifying combos go in the sitemap.
• Results: the exact same venue-card partial as /venues, pre-filtered to event_type + emirate; "View all" links to the full filtered listing. Each card carries both View details and Enquire.
• Intro + FAQ are templated per event×city from data. The FAQ also emits FAQPage JSON-LD structured data for rich results.
• Internal links are auto-generated: same event in sibling emirates, other events in this city, plus All event types / All locations — strong internal linking for SEO.
• Head: unique <title> + meta description + canonical (from U5-a). Imagery is the real venue photos (gradients here are placeholders).
