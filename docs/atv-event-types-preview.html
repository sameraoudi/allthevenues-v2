How this page behaves
• Featured cards (6): Wedding, Corporate, Conference, Product Launch, Private Party, Exhibition — each a full-bleed image tile that links to the matching /venues?event_type=… filter.
• Gated cards (2): Gala Dinner & Yacht Event render only when they have ≥1 published venue. Until you tag venues, they stay hidden from the public page (the dashed state above is a build-note, not what visitors see). No code change needed to switch them on — just tag venues.
• Enquiry bridges: a slim bar under the intro ("Already know what you need?") for planners who don't want to browse, plus the "Not sure which fits?" strip at the bottom — both route into the assisted enquiry flow.
• Venue counts (rule): a count renders only when it's reasonably strong (build threshold ≈ ≥5 venues; tunable). Below that — and for a just-activated Gala/Yacht — the pill reads "Explore venues" instead of a weak number, so no category looks thin. Numbers shown here are illustrative.
• Hover: the approved card lift + a slow image zoom. On the four smaller tiles the descriptor fades in on hover (desktop); mobile shows headings only to avoid crowding.

Image spec (for the photos you're preparing)
• Aspect 3:2 landscape, min 1600px wide, exported WebP (re-encoded on upload anyway).
• Keep the lower-left third calmer — that's where the name + descriptor sit over a dark scrim.
• Subject weighted center/right; consistent light + mood across all six so the grid reads as one set.
• One image per featured type (6 now; +2 for Gala/Yacht when you're ready).

Not on this page (still available as /venues filters): Meeting, Training, Networking, Engagement, Birthday, Outdoor Event, Other.
