# All The Venues Visual Specification

## 1. Purpose

This document defines the approved visual direction for the revamped **All The Venues** platform.

The approved look and feel is:

**Coastal UAE**

This direction should guide UI design, branding, front-end implementation, image selection, component styling, and content presentation across the public website, enquiry flow, admin experience, and partner portal.

---

## 2. Design Intent

All The Venues should feel like a premium UAE venue discovery platform, not a generic directory or dry SaaS product.

The visual identity should combine:

- Premium hospitality
- Coastal UAE freshness
- Editorial venue discovery
- Trustworthy lead-generation functionality
- Warm but modern user experience
- Practical search and enquiry flows

The site should feel:

- Elegant
- Calm
- Visual
- Human
- Curated
- Trustworthy
- Easy to use
- Commercially credible

The platform should not feel:

- Generic
- Overly corporate
- Template-generated
- Cold or dry
- Overly gold/luxury cliché
- Cluttered
- Like a classified ads website

---

## 3. Approved Creative Direction

## 3.1 Style Name

**Coastal UAE**

## 3.2 Design Summary

A fresh, premium, image-led visual system inspired by UAE coastal venues, luxury hospitality, refined event planning, and modern digital platforms.

The interface should use deep navy, pearl backgrounds, muted teal CTAs, champagne accents, high-quality venue photography, elegant spacing, and a clean editorial layout.

## 3.3 Design Statement

**A premium UAE venue guide with the usability of a modern lead-generation platform.**

The visual design should create confidence and desire, while still making venue search, comparison, and enquiry very easy.

## 4. Color Palette

## 4.1 Approved Core Palette

| Role | Name | Hex | Usage |
|---|---|---:|---|
| Primary dark | Deep Navy | `#0E1B2A` | Header, footer, dark panels, premium contrast |
| Main background | Pearl | `#F7F4EF` | Page background, calm warm base |
| Surface | White | `#FFFFFF` | Cards, forms, content panels |
| Soft section | Mist Blue | `#DCE8E8` | Light section backgrounds, subtle coastal tone |
| Primary CTA | Muted Teal | `#1F6F78` | Primary buttons, active filters, key actions |
| Premium accent | Champagne | `#C9A76A` | Featured badges, small highlights, icons |
| Text | Charcoal Navy | `#1C2A36` | Main text and headings |
| Muted text | Slate Grey | `#66717A` | Secondary text, metadata, helper text |

## 4.2 Extended Support Colors

| Role | Name | Hex | Usage |
|---|---|---:|---|
| Border | Soft Stone | `#E4DED5` | Card borders, input borders, dividers |
| Light teal surface | Pale Aqua | `#EEF6F6` | Filter panels, trust blocks, subtle alerts |
| CTA hover | Deep Teal | `#155A63` | Button hover state |
| Navy hover | Midnight Navy | `#07111D` | Footer/header hover depth |
| Warm highlight | Light Champagne | `#EFE2C4` | Soft highlight backgrounds |
| Error | Muted Red | `#B4534A` | Form validation errors |
| Success | Sea Green | `#2F7D68` | Successful submission states |

## 4.3 Color Usage Rules

- Use **Deep Navy** for the strongest contrast areas, especially footer, partner CTA panels, and some navigation states.
- Use **Pearl** as the main page background instead of pure white, to avoid a cold SaaS feeling.
- Use **White** for cards and form surfaces.
- Use **Muted Teal** for primary actions such as:
  - Find venues
  - Enquire now
  - Submit enquiry
  - Apply filters
  - Partner portal actions
- Use **Champagne** sparingly:
  - Featured badges
  - Small icons
  - Premium accents
  - Section highlights
- Do not overuse Champagne as a dominant background, because it can make the site feel fake luxury.
- Avoid bright blue, neon green, heavy black, and loud gradients.

## 5. Typography

## 5.1 Recommended Typography Pairing

Recommended pairing:

- **Headings:** Cormorant Garamond or Playfair Display
- **Body/UI:** Manrope or Inter

Preferred final pairing:

- **Cormorant Garamond** for large editorial headings
- **Manrope** for body text, navigation, buttons, forms, cards, filters, and dashboards

## 5.2 Typography Usage

| Element | Font | Weight | Notes |
|---|---|---|---|
| Hero heading | Cormorant Garamond | 500 or 600 | Large, elegant, editorial |
| Section headings | Cormorant Garamond | 500 or 600 | Use selectively |
| Venue titles | Manrope or Cormorant Garamond | 600 | Keep readable in cards |
| Body text | Manrope | 400 | Clear and modern |
| Navigation | Manrope | 500 | Small, clean, spaced |
| Buttons | Manrope | 600 | Confident and readable |
| Form labels | Manrope | 500 | Practical and clean |
| Metadata | Manrope | 400 or 500 | Small but legible |

## 5.3 Typography Scale

Recommended desktop scale:

| Element | Size |
|---|---:|
| Hero heading | 56 to 72 px |
| Page title | 40 to 52 px |
| Section heading | 30 to 40 px |
| Card title | 18 to 22 px |
| Body | 15 to 17 px |
| Small text | 13 to 14 px |
| Labels | 12 to 14 px |

Recommended mobile scale:

| Element | Size |
|---|---:|
| Hero heading | 38 to 46 px |
| Page title | 32 to 38 px |
| Section heading | 26 to 32 px |
| Card title | 18 to 20 px |
| Body | 15 to 16 px |
| Small text | 13 px |

## 5.4 Typography Rules

- Use the serif font only for selected headings and editorial moments.
- Do not use serif typography for forms, filters, or dense UI.
- Avoid overly decorative fonts.
- Keep line height generous:
  - Headings: 1.05 to 1.15
  - Body: 1.55 to 1.7
- Use sentence case for buttons and labels.
- Avoid all-caps except for small badges.

## 6. Imagery Direction

## 6.1 Image Style

Photography is central to the brand. The site should rely on real venue photography wherever possible.

Preferred imagery:

- UAE waterfront venues
- Hotel ballrooms
- Outdoor garden events
- Rooftops
- Beachfront settings
- Yacht venues
- Elegant table settings
- Evening hospitality scenes
- Architectural details
- Real venue interiors

Image mood:

- Warm
- Atmospheric
- Premium
- Real
- Elegant
- Calm
- Slightly cinematic

## 6.2 Avoid

Avoid:

- Generic AI-generated event scenes
- Fake luxury interiors
- Overused stock photos
- Smiling business people with no context
- Overly staged wedding stock imagery
- Heavy filters
- Clashing photo styles
- Low-resolution venue images
- Images with watermarks

## 6.3 Image Treatment

Recommended treatment:

- Use large image-led cards.
- Use soft overlays on hero images for readability.
- Keep image corners slightly rounded.
- Use consistent crop ratios.
- Avoid excessive drop shadows.
- Use image gradients only when needed for text readability.

Recommended ratios:

| Use | Ratio |
|---|---|
| Homepage hero | 16:9 or wide cinematic |
| Venue cards | 4:3 or 3:2 |
| Venue detail hero | 16:9 |
| Gallery thumbnails | 4:3 |
| Partner CTA image | 3:2 |
| Blog/landing page cards | 16:10 or 4:3 |

## 7. Layout System

## 7.1 Overall Layout Character

The layout should feel:

- Spacious
- Editorial
- Structured
- Search-led
- Image-led
- Calm and refined

Avoid making every section look like the same grid. Mix:

- Full-width hero areas
- Floating search panels
- Editorial featured venue sections
- Practical listing grids
- Trust blocks
- Partner CTA bands
- Clean form cards
- Structured dashboard panels

## 7.2 Grid

Recommended desktop grid:

- Max content width: 1180 to 1280 px
- Page gutters: 32 to 48 px
- Card gap: 24 px
- Section spacing: 72 to 112 px
- Form/card padding: 24 to 40 px

Recommended mobile:

- Page gutters: 16 to 20 px
- Card gap: 16 px
- Section spacing: 48 to 72 px
- Form/card padding: 18 to 24 px

## 7.3 Border Radius

Use refined, moderate rounding.

| Component | Radius |
|---|---:|
| Buttons | 8 to 10 px |
| Cards | 12 to 16 px |
| Inputs | 8 to 10 px |
| Image cards | 12 to 16 px |
| Large panels | 16 to 20 px |
| Badges | 999 px or 6 px depending on style |

Avoid excessive pill-shaped UI everywhere.

## 7.4 Shadows

Use subtle shadows only.

Recommended card shadow:

```css
box-shadow: 0 12px 32px rgba(14, 27, 42, 0.08);
```

Recommended floating search shadow:

```css
box-shadow: 0 18px 48px rgba(14, 27, 42, 0.16);
```

Avoid heavy, dark, or obvious template shadows.

## 15. Microcopy and Content Style

## 15.1 Copy Direction

Copy should feel specific, helpful, and human.

Use:

- “Find venues that match your event.”
- “Tell us what you are planning.”
- “Enquire with selected venues.”
- “Partner with All The Venues.”
- “Curated and reviewed venue information.”
- “Up-to-date capacity, facilities, and minimum spend details.”

Avoid:

- “Unlock seamless experiences.”
- “Elevate your journey.”
- “Reimagine event planning.”
- “World-class solutions.”
- “Dream venues await.”
- “One-stop destination” unless the feature set supports it.

## 15.2 Venue Description Style

Venue descriptions should be concise, specific, and useful.

Good example:

**A waterfront garden venue suited for weddings, private dinners, and cocktail receptions, with skyline views and flexible outdoor seating.**

Weak example:

**An amazing venue that offers unforgettable experiences for all your special moments.**

## 16. Responsive Behavior

## 16.1 Mobile Priorities

Mobile users should be able to:

- Understand the site immediately.
- Start a search quickly.
- Browse venue cards comfortably.
- Open filters easily.
- Submit an enquiry without friction.
- Contact or enquire from venue detail pages.

## 16.2 Mobile Layout Rules

- Hero text should remain readable.
- Search panel should stack vertically.
- Venue cards should use full-width stacked layout.
- Filter sidebar should become drawer or accordion.
- Enquire Now button should remain easy to access.
- Avoid oversized images that push content too far down.
- Keep forms short and progressive.

---

## 17. Accessibility Requirements

Minimum visual accessibility rules:

- Text must have sufficient contrast.
- Buttons must have clear hover and focus states.
- Form fields must have visible labels.
- Error messages must be clear and close to the related field.
- Do not communicate meaning by color alone.
- All important images need alt text.
- Keyboard navigation must be visible and usable.
- Font sizes should remain readable on mobile.

## 19. Do and Do Not Summary

## 19.1 Do

- Use real venue photography.
- Keep the interface spacious.
- Use Muted Teal for core actions.
- Use Champagne sparingly.
- Make search and enquiry highly visible.
- Use editorial headings.
- Keep cards image-led.
- Use specific human copy.
- Preserve a premium but practical feel.
- Make the partner portal feel professional.

## 19.2 Do Not

- Do not make the platform look like a generic SaaS dashboard.
- Do not use too many icons.
- Do not rely on AI-looking imagery.
- Do not use heavy gradients.
- Do not overuse gold.
- Do not make all sections identical grids.
- Do not hide the enquiry CTA.
- Do not use vague marketing copy.
- Do not make the interface feel like a classified ads site.
- Do not allow partner content to look unreviewed or inconsistent.

---

## 20. Final Visual Direction Statement

The approved visual direction for All The Venues is:

**Coastal UAE: a premium, image-led, editorial venue discovery experience using deep navy, pearl backgrounds, muted teal actions, champagne accents, refined typography, and real UAE venue photography.**

The platform should feel elegant and trustworthy while remaining practical, searchable, and conversion-focused.
