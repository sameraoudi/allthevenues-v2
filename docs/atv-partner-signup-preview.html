How this page works
• Route: /become-a-venue-partner. The navbar "Become a Venue Partner" CTA (currently mis-pointed at the venue enquiry form) is repointed here.
• Storage (structured, not notes-packed): saved as an enquiry with mode = 'partner_signup' (new enum, #2-1) so it lands in the existing admin lead inbox, clearly separated from event enquiries. Mapping: org → company, contact → name, email/ phone direct, primary location → city_pref (free text — handles Al Ain, which isn't an emirate), message → notes, checkbox → consent_to_share; and three new dedicated columns provider_type / website / venues_managed so the data supports future provider reporting/onboarding.
• Inbox: the admin enquiry detail surfaces these partner fields for partner_signup rows, and the inbox can filter by this mode.
• Hero: a soft venue image sits under the navy gradient (reusing the homepage hero treatment) for warmth — restrained, not busy. Value cards keep overlapping the hero.
• Security: reuses the enquiry form's CSRF + Turnstile + rate-limit; admin gets a mail notification. No public PII.
• Provider types add Yacht provider and Other to the existing Hotel/Resort/Restaurant/ Unique venue set.
• The Venue Provider Terms + Privacy links point to pages built in backlog item #7 (legal) — stubs until then. Styling reuses the site's form components (real build uses brand.css, no CDN).
